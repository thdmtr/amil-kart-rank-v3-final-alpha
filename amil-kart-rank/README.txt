#################################################################
Instruções para a equipe de TI da AMIL Corp.
#################################################################

1 - Um JAR compilado executável está disponível na pasta dist do projeto. O caminho para o
arquivo que é solicitado logo no começo, deve ser absoluto, exemplo: E:\sample-data.txt

2 - Os testes unitários encontram-se na package com.amil.junit.tests, bastando executar, sem nada à configurar.

3 - O programa foi desenvolvido de tal maneira que o arquivo bruto fornecido de dados é automaticamente tratado, não é necessária
nenhuma alteração ou preparação, conforme solicitado.

4 - A interface é bem simples e objetiva, contempla 90% das solicitações, fiz em J2SE porque é muito mais ágil, sem contar que acabei
de sair de um projeto J2SE puro(PDV novo da Drogasil).

5 - Para executar, somente frisando, basta dar dois cliques no jar fornecido na pasta dist do projeto.

6 - Qualquer crítica é sempre muito bem vinda, afinal estou sempre disposto à evoluir/melhorar. Já aviso que sou péssimo em design(rs).
